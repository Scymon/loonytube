## New findings

### P1 — Download endpoint will timeout on Vercel (no `maxDuration`)

**File:** `src/app/api/videos/[id]/download/route.ts`, lines 36–50

The route polls Cloudflare for an MP4 download URL in a 60-second blocking loop (2 s sleep between polls). No `export const maxDuration` is set in the route and there is no `vercel.json`. Vercel's default max execution for Node.js functions is **10 seconds** on all plans without explicit configuration. On Vercel Pro the ceiling is 60 s but still requires `maxDuration = 60` to be declared.

In practice: the first time a user clicks Download, Cloudflare generates the MP4 asynchronously (CF docs say typically <30 s). The server will almost certainly hit the 10 s Vercel deadline before the MP4 is ready, and the client receives a 504 gateway error (not the graceful 202 the code intends to return). The button stays in "Preparing…" state with no user feedback.

**Two viable fixes:**
1. **Quick fix (S):** Add `export const maxDuration = 60;` to the route file. Requires Vercel Pro. Covers the common case.
2. **Proper fix (M):** Make the endpoint non-blocking. POST immediately triggers CF generation and returns 202. Client polls `GET /api/videos/[id]/download/status` until the URL is ready, then triggers the download.

**Impact:** The download feature is functionally broken for most users on the first attempt. Creator trust — owners expect to download their own content.

**Size:** S (quick) or M (proper). **Status:** New. **Priority: P1.**

### P2 — Arrow keys navigate (prev/next) rather than seek — diverges from expectation

**File:** `src/hooks/useKeyboardShortcuts.ts`, lines 66–87

`ArrowLeft` → previous video in queue; `ArrowRight` → next video. YouTube and most video players use `←`/`→` to seek ±5 seconds. This platform uses `j`/`l` for ±10 s seek (correct) but has no 5 s seek shortcut at all. Users who press `←` expecting to rewind will instead navigate away from the video.

DashHero adds its own `keydown` listener with the same key mapping for within-channel navigation. Both listeners coexist on the document without conflict (different pages) but the design is consistent: arrow keys = navigate, not seek.

This is a deliberate "channel surfing" design choice. It is worth noting because it departs from web video conventions; the OSD ("▶ Play", "⏸ Pause") does not currently announce seek actions initiated by arrow keys, only by J/L.

**Requires manual testing** to assess whether users find it intuitive.

**Fix (optional):** Either add a 5 s seek shortcut (e.g., hold Shift+← / Shift+→) or document the shortcut scheme in a help tooltip. For MVP this is a refinement, not a blocker.

**Size:** S. **Status:** New (low priority).
